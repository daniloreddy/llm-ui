Guida all'uso

LLM UI è un'interfaccia web per chattare con modelli linguistici locali o remoti compatibili con l'API OpenAI.

Configurare un endpoint

Vai in Config → + Aggiungi per creare un nuovo endpoint. I campi disponibili:

CampoDescrizione
NomeEtichetta visualizzata nell'intestazione del pannello
Server URLURL base del server LLM (es. http://127.0.0.1:8080)
URL completoSpunta se l'URL punta già all'endpoint esatto (es. Cloudflare AI Gateway)
ModelIdentificatore del modello (es. gpt-4o). Lascia vuoto per usare il default del server
API KeyBearer token di autenticazione (opzionale)
System PromptIstruzione di sistema preposta ad ogni conversazione
TemperatureCasualità delle risposte (0 = deterministico, 2 = molto creativo)
Top-PNucleus sampling — filtra i token meno probabili
Max TokensLunghezza massima della risposta. 0 = nessun limite
Repeat PenaltyPenalità per ripetizioni nel testo generato

Usa ⎘ per clonare un endpoint (copia tutti i campi inclusa la chiave API) e ⠿ per riordinarlo trascinandolo.

Preferenze

La sezione Preferenze in Config permette di personalizzare l'interfaccia:

  • Font chat — scegli tra i font di sistema o un Google Font preimpostato (Inter, Roboto, Source Sans 3, Lato, JetBrains Mono). Con Personalizzato puoi digitare il nome di qualsiasi font Google.
  • Lingua — Italiano o English. Il cambio si applica immediatamente a tutta l'interfaccia e alla guida.

Layout pannelli

La sezione Layout pannelli in Config permette di affiancare fino a 4 endpoint contemporaneamente. Scegli il numero di pannelli (1–4) e assegna un endpoint a ciascuno tramite i menu a tendina. Il layout a 3 mostra un pannello grande a sinistra e due più piccoli a destra.

Modalità input

Il pulsante in alto a destra nella barra di navigazione controlla come funziona l'input:

  • ⇥ Per pannello (default) — ogni pannello ha la propria casella di testo. Puoi fare domande diverse a ogni modello nello stesso momento.
  • ⇉ Broadcast — un unico campo in basso invia lo stesso messaggio a tutti i pannelli attivi in parallelo. Utile per confrontare le risposte di modelli diversi alla stessa domanda.

La scelta viene salvata automaticamente.

Inviare messaggi

  • Invio — invia il messaggio
  • Shift + Invio — va a capo senza inviare
  • Pulsante ➤ — invia con il mouse

Cronologia dei prompt

Nella casella di testo, usa ↑ e ↓ per navigare i messaggi inviati in precedenza (come nel terminale). La bozza corrente viene ripristinata premendo ↓ fino alla fine della cronologia. In modalità broadcast la cronologia è condivisa; in modalità per pannello è indipendente per ciascun pannello.

Allegati

Clicca 📎 per allegare file. Tipi supportati:

  • Testo (.txt, .md, .json, .py, .js, ecc.) — il contenuto viene aggiunto al messaggio
  • PDF — il testo viene estratto nel browser e allegato al messaggio
  • Immagini — inviate come messaggi multimodali (richiede un modello vision)

In modalità per pannello ogni pannello gestisce i propri allegati in modo indipendente.

Lettura delle risposte

Le risposte dell'assistente sono rese in Markdown con evidenziazione della sintassi.

  • Copia codice — ogni blocco di codice ha un pulsante ⎘ in alto a destra per copiare il contenuto negli appunti.
  • Blocchi di ragionamento — se il modello emette contenuto in tag <think> o <thinking>, viene mostrato in un accordion ⚙ Ragionamento collassabile, separato dalla risposta principale.

Rigenerare una risposta

Sotto ogni risposta dell'assistente compare il pulsante ↺ Rigenera. Cliccalo per rinviare l'ultimo messaggio utente e ottenere una nuova risposta dallo stesso modello, senza riscrivere il prompt. La risposta precedente viene rimossa dalla cronologia.

Contatore token

Accanto alla casella di input compare una stima del numero di token nella cronologia corrente (es. ~1.2K tk). È una stima approssimativa (4 caratteri ≈ 1 token) utile per tenere d'occhio la finestra di contesto.

Interrompere la generazione

Clicca ⏹ durante lo streaming per interrompere la generazione. Il testo già ricevuto viene mantenuto nella chat e nella cronologia.

Pulire la chat

  • ↺ nell'intestazione del pannello — cancella la cronologia del singolo pannello
  • ↺ nella barra in basso (modalità broadcast) — cancella tutti i pannelli

Esportare la chat

Clicca ⬇ nell'intestazione del pannello e scegli il formato:

  • Markdown — file .md leggibile con qualsiasi editor
  • JSON — struttura completa con metadati endpoint, utile per elaborazioni successive

Console HTTP

Clicca >_ nell'intestazione del pannello per aprire il drawer di log. Mostra i dettagli di ogni richiesta: parametri usati, status HTTP ricevuto e numero di chunk dello stream. Utile per diagnosticare problemi di connessione.

Esportare / importare la configurazione

In Config → Esporta scarichi un file JSON con tutti gli endpoint configurati. Con Importa puoi ripristinare la configurazione su un altro dispositivo o dopo una reinstallazione.
