User Guide

LLM UI is a web interface for chatting with local or remote language models compatible with the OpenAI API.

Configuring an endpoint

Go to Config → + Add to create a new endpoint. Available fields:

FieldDescription
NameLabel displayed in the panel header
Server URLBase URL of the LLM server (e.g. http://127.0.0.1:8080)
Full URLCheck if the URL already points to the exact endpoint (e.g. Cloudflare AI Gateway)
ModelModel identifier (e.g. gpt-4o). Leave blank to use the server default
API KeyBearer authentication token (optional)
System PromptSystem instruction prepended to every conversation
TemperatureResponse randomness (0 = deterministic, 2 = very creative)
Top-PNucleus sampling — filters out less probable tokens
Max TokensMaximum response length. 0 = no limit
Repeat PenaltyPenalty for repetitions in the generated text

Use ⎘ to clone an endpoint (copies all fields including the API key) and ⠿ to reorder it by dragging.

Panel layout

The Panel layout section in Config lets you display up to 4 endpoints side by side. Choose the number of panels (1–4) and assign an endpoint to each using the dropdown menus. The 3-panel layout shows one large panel on the left and two smaller ones on the right.

Input mode

The button in the top-right corner of the navigation bar controls how input works:

  • ⇥ Per panel (default) — each panel has its own text box. You can ask different questions to each model at the same time.
  • ⇉ Broadcast — a single input field at the bottom sends the same message to all active panels in parallel. Useful for comparing different models' responses to the same question.

The choice is saved automatically.

Sending messages

  • Enter — send the message
  • Shift + Enter — new line without sending
  • ➤ button — send with the mouse

Attachments

Click 📎 to attach files. Supported types:

  • Text (.txt, .md, .json, .py, .js, etc.) — content is appended to the message
  • PDF — text is extracted in the browser and attached to the message
  • Images — sent as multimodal messages (requires a vision model)

In per-panel mode, each panel manages its own attachments independently.

Stopping generation

Click ⏹ during streaming to stop generation. Text already received is kept in the chat and history.

Clearing the chat

  • ↺ in the panel header — clears the history of that single panel
  • ↺ in the bottom bar (broadcast mode) — clears all panels

Exporting the chat

Click ⬇ in the panel header and choose the format:

  • Markdown — .md file readable with any editor
  • JSON — full structure with endpoint metadata, useful for further processing

HTTP Console

Click >_ in the panel header to open the log drawer. It shows details of each request: parameters used, HTTP status received, and the number of stream chunks. Useful for diagnosing connection issues.

Exporting / importing configuration

In Config → Export you download a JSON file with all configured endpoints. With Import you can restore the configuration on another device or after a reinstall.
